# Интеграции с внешними системами (roadmap)

Краткий продуктовый ориентир без реализации в коде. Цель — снизить трение внедрения в школе и у репетиторов с уже привычными инструментами.

## Приоритетные направления

1. **Экспорт оценок**  
   Уже есть экспорт XLSX из интерфейса. Следующий шаг — опциональная выгрузка в формат, удобный для импорта в электронные журналы (если появится стабильный формат у целевой аудитории).

2. **Ссылка и QR как «минимальная интеграция»**  
   Для многих классов достаточно стабильной выдачи задания по коду и ссылке без API. Документировать сценарий «урок за 2 минуты» в инструкции для учителя.

3. **LMS и классные комнаты**  
   - Google Classroom: добавление задания по ссылке, список учеников — требует OAuth и отдельного backend или Firebase Cloud Functions.  
   - Отечественные экосистемы (МЭШ и др.): зависят от доступа к API и юридических ограничений; до тех пор — ручной обмен ссылкой и файлом результатов.

4. **SSO / вход через школу**  
   Долгий горизонт: SAML/OAuth школьного провайдера. Имеет смысл после стабилизации ролей и модели данных.

## Рекомендация

Не начинать полноценный OAuth/LMS до того, как в продукте зафиксированы: типы вопросов, политика попыток, теги/темы и стабильная схема `decks` / `results`. Иначе интеграции придётся переписывать.
