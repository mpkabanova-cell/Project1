# Продукт: интерактивные карточки (MVP по обновлённому ТЗ)

Ориентир по механике для учителя и ученика. Текущий прототип — один файл [`index.html`](../index.html), Firebase (Firestore + Auth).

## Фокус MVP

- **Учитель**: один экран «рабочий стол» — ввод текста → генерация карточек → inline-редактирование → «Выдать» (ссылка + код) → компактная аналитика по этому набору.
- **Ученик**: отдельный поток входа по ссылке `#q=…` или коду, без обязательной регистрации там, где это уже реализовано в приложении.
- **Типы карточек (внутренняя модель)**: `input` (свободный ввод, эталоны в `accepted`), `choice` (ровно 4 варианта + индекс верного), `flashcard` (вопрос / ответ, самооценка), `matching` (пары).
- Старые типы (`single`, `multi`, `open`, `tf`, `match`) при загрузке и в прохождении приводятся к этой модели через слой совместимости.

## Что сознательно не в главном потоке MVP

Отдельная библиотека как главный вход, экран создания задания «как раньше», экран задания с вкладками «Карточки / Пройти / Результаты», глобальная статистика и экспорт XLSX — код в файле может частично сохраняться, но сценарий учителя завязан на **рабочий стол**. Глобальная статистика не предлагается как основной путь.

## Данные в Firestore

- Коллекция **`decks`**: наборы учителя. Новые наборы с рабочего стола сохраняются с **`schemaVersion: 2`**, **`mode: 'mixed'`**, карточки в формате MVP (`type`, поля по типу).
- Коллекция **`results`**: прохождения; для мини-аналитики используются поля вроде `deckId`, `pct`, массив **`answers`** с привязкой к карточке (`cardId`, `correct`).

## Сжатая ссылка `#quiz=…` (без облака)

Для локального/демо-режима набор можно передавать в URL. Формат пакета поддерживает **`sv: 2`** и компактное кодирование вариантов выбора (`os` + `ci`) и эталонов для `input` (`ac`).

## Дальнейшее развитие (вне MVP)

Расширяемые типы (`multi`, `sequence`, `image`), продвинутые настройки теста, папки/теги/архив, отчёты по всем заданиям — по мере необходимости, с сохранением обратной совместимости для уже сохранённых `decks`.
